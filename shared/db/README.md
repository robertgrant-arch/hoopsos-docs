# shared/db

Drizzle ORM schema and Neon Postgres client for HoopsOS.

## Layout

- `schema/` Drizzle table definitions, one file per logical module.
- `client.ts` Neon serverless client factory.
- `index.ts` Barrel export. Import as `import { db, schema } from "@shared/db"`.
- `migrations/` SQL generated by drizzle-kit. Do not edit by hand.

## Environment variables

- `DATABASE_URL` Pooled Neon connection string. Used at runtime.
- `DATABASE_URL_UNPOOLED` Direct connection string. Used by drizzle-kit for migrations.
- `FILM_ANALYSIS_DATA_SOURCE` `mock` (default) or `db`. Keep `mock` until the schema has been migrated.

Set all three in Vercel for Development, Preview, and Production.

## Local migration workflow

    pnpm install
    pnpm db:generate
    pnpm db:migrate

## Tenancy

Every domain table includes `org_id uuid not null` referencing `orgs.id`. Code outside this folder should go through the repository layer which enforces tenant scoping. PG RLS policies deferred to PR 6.

## Conventions

- ids are `uuid` via `gen_random_uuid()`
- timestamps are `timestamptz` with `default now()`
- every table has `created_at`, `updated_at`, nullable `deleted_at`
- status fields are pg enums declared in `schema/_enums.ts`
- provider-specific extras live in a `payload jsonb` column
